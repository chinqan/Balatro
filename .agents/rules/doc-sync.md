# 文件同步規則

當對遊戲進行任何修改時，若該修改 **不是 Bug 修正**，則必須同步更新以下文件：

1. **`docs/gdd/`** — 遊戲設計規格文件（GDD 相關文件）
2. **`docs/plan/`** — 計劃書與工作流文件

## 判斷標準

- **Bug 修正**：僅修正程式碼中的錯誤行為，使其符合現有規格。這類修改 **不需要** 更新文件。
- **非 Bug 修正**（功能新增、機制調整、數值變更、UI/UX 改動等）：**必須** 同步更新對應的 `docs/gdd/` 規格文件與 `docs/plan/` 計劃書，確保文件始終反映最新的遊戲設計與開發計劃。

## 更新範圍

- 若改動涉及遊戲機制、數值、流程等設計層面，更新 `docs/gdd/` 中對應的規格文件。
- 若改動影響開發進度、任務排程或工作流，更新 `docs/plan/` 中對應的計劃書與 workflow 文件。
- 更新內容應準確描述變更，保持文件與實際實作一致。
